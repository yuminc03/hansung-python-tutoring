# 🚑 GitHub Pages 404 에러 해결 가이드 (React + Vite)

Vite로 구축한 React 웹사이트를 GitHub Pages에 배포했을 때 **404 오류**가 발생하는 주요 원인과 해결 방법을 나중에도 쉽게 찾아볼 수 있도록 정리한 기록입니다.

---

## 1. 웹사이트 자체가 접속되지 않고 404가 뜰 때 (설정 누락)
*   **원인:** 터미널에서 `npm run deploy`를 쳐서 빌드 결과물을 `gh-pages` 브랜치에 무사히 업로드했더라도, GitHub 웹사이트 자체에서 "이 브랜치를 화면에 보여줘!"라고 허락(설정)해주지 않으면 접속이 불가능합니다.
*   **해결 방법:**
    1. GitHub 저장소의 **Settings (설정)** -> 좌측 **Pages** 메뉴로 이동합니다.
    2. **Build and deployment** 섹션의 `Source`를 **Deploy from a branch**로 맞춥니다.
    3. `Branch` 설정을 기존(`None` 또는 `main`)에서 **`gh-pages`**로 변경하고 **[Save]**를 누릅니다. (반영까지 1~2분 소요)

## 2. 화면이 하얗게 나오거나 리소스를 못 찾을 때 (경로 불일치)
*   **원인:** GitHub Pages는 `https://아이디.github.io/저장소이름/` 형태의 주소를 사용합니다. 하지만 React는 기본적으로 웹사이트의 최상단 루트(`/`)를 기준으로 파일을 찾으려 하기 때문에 엉뚱한 경로에서 파일을 찾다가 404 에러가 납니다.
*   **해결 방법:**
    *   `vite.config.js` 파일에 저장소 이름과 똑같은 **`base` URL**을 명시해 주어야 합니다.
    *   ```javascript
        export default defineConfig({
          base: '/hansung-python-tutoring/', // 저장소 이름 앞뒤로 슬래시(/) 필수
          plugins: [react()],
        })
        ```

## 3. 새로고침을 하거나 직접 링크를 치고 들어갈 때 404가 뜰 때 (라우팅 방식)
*   **원인:** 일반적인 `BrowserRouter`를 사용하면, 사용자가 `주소/note/1`로 접속할 때 GitHub 서버는 실제로 `note`라는 폴더를 찾으려 시도하다가 없으므로 404 에러를 뿜어냅니다. (서버가 React보다 먼저 개입하기 때문입니다.)
*   **해결 방법:**
    *   `App.jsx`에서 `BrowserRouter`를 **`HashRouter`**로 변경합니다.
    *   주소에 `/#/`이 붙어 보기에는 조금 다를 수 있지만, `#` 뒤의 주소는 서버로 넘어가지 않고 브라우저(React)에서만 처리되므로 404 에러를 원천 차단할 수 있습니다.

## 4. 알 수 없는 이유로 화면이 깨지거나 파일이 누락될 때 (Jekyll 무시 현상)
*   **원인:** GitHub Pages는 내부적으로 'Jekyll'이라는 시스템을 돌리는데, 이 시스템의 규칙상 파일명 앞에 밑줄(`_`)이나 점(`.`)이 들어가면 "이건 화면에 띄울 파일이 아니야"라고 판단해 무시해버립니다. Vite가 만들어낸 파일 구조와 충돌할 때가 있습니다.
*   **해결 방법:**
    *   우리가 만든 빌드 폴더(`dist`) 안에 **`.nojekyll`**이라는 빈 파일을 하나 만들어주면, GitHub가 "아, 이 사이트는 Jekyll 시스템을 안 쓰는구나!"하고 모든 파일을 정상적으로 서빙합니다.
    *   `package.json`의 스크립트를 수정하여 빌드할 때마다 자동으로 생성되게 합니다.
    *   `"predeploy": "npm run build && touch dist/.nojekyll"`
